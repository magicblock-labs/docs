## 取り組む issue を探す

まず始めたい場合は、[公開されている issues](https://github.com/magicblock-labs/Solana.Unity-SDK/issues) を確認してください。簡単なドキュメント修正や誤字の修正であれば、そのまま pull request を作成して構いません。

より大きな変更や自発的な新機能を検討している場合は、まず issue を作成し、[Telegram](https://t.me/+78KHQkUsy0ViMzQ6) でメンテナーとやり取りしてください。

## issue を選ぶ

貢献したい場合は、コメントし、fork し、たとえ空でも pull request を作成して issue を担当することを示してください。これにより、メンテナーは誰がどの issue に取り組んでいるかを把握でき、作業の重複を防げます。

## ガイドライン

以下のガイドラインに従ってください。

コーディング前：

- 取り組んでいる issue を表すブランチ名を付ける

コーディング中：

- できるだけ早く draft PR を提出する
- PR に直接関係するコードだけを変更すること。大きくリファクタリングしたくなるコードを見つけることもあるかもしれませんが、PR に関係がなければ手を入れず、代わりに issue を作成してください。これにより、レビュアーは一度に一つの問題に集中できます。
