## Anchor プログラムを書く

このカウンタープログラムの主要な構成要素を見ていきましょう。

## コア機能

このプログラムは主に 2 つの instruction を実装しています。

1. `initialize`: カウンターを 0 に設定する
2. `increment`: カウンターを 1 増やす

以下がプログラムの基本構造です。

```rust
#[ephemeral]
#[program]
pub mod anchor_counter {
    use super::*;

    /// Initialize the counter.
    pub fn initialize(ctx: Context<Initialize>) -> Result<()> {
        let counter = &mut ctx.accounts.counter;
        counter.count = 0;
        Ok(())
    }

    /// Increment the counter.
    pub fn increment(ctx: Context<Increment>) -> Result<()> {
        let counter = &mut ctx.accounts.counter;
        counter.count += 1;
        Ok(())
    }

    // ... Additional instructions will be added here
}
```

ここに特別なことはなく、単純にカウンターを増やす Anchor プログラムです。違いは、delegation program とやり取りするための便利なロジックを注入する `delegate` macro を追加している点だけです。

### Counter PDA の委任

counter PDA を委任し、Ephemeral Rollup セッション内で writable にするためには、内部で `delegate_account` 関数を呼び出す instruction を追加する必要があります。`delegate_account` は delegation program に CPI を行い、検証が通るとそのアカウントの所有権を得ます。
このステップの後、ephemeral validator は counter PDA 上でトランザクションを処理し、delegation program を通じて状態差分を提案できるようになります。

### PDA が委任中のままコミットする

ephemeral runtime では、PDA が委任されたままでもその状態をコミットできます。これは `commit_accounts` 関数を呼び出して行います。

```rust
/// Increment the counter and manually commit the account in the Ephemeral Rollup session.
pub fn increment_and_commit(ctx: Context<IncrementAndCommit>) -> Result<()> {
    let counter = &mut ctx.accounts.counter;
    counter.count += 1;
    commit_accounts(
        &ctx.accounts.payer,
        vec![&ctx.accounts.counter.to_account_info()],
        &ctx.accounts.magic_context,
        &ctx.accounts.magic_program,
    )?;
    Ok(())
}
````

### PDA の委任解除

PDA の委任解除は、ある instruction の一部として `commit_and_undelegate_accounts` を呼ぶことで行います。
undelegation は最新の状態をコミットし、PDA の所有権を owner program に戻します。

### Ephemeral エンドポイント設定

Ephemeral Rollup セッションとやり取りするには、適切なエンドポイントを設定する必要があります。

- devnet では以下の ephemeral エンドポイントを使います。

  https://devnet.magicblock.app

- mainnet については、適切なエンドポイントを受け取るため MagicBlock チームにお問い合わせください。
- <a href="/jp/pages/get-started/how-integrate-your-program/local-setup">localhost</a> の場合は、適切な環境変数で ephemeral validator をローカルにインストールして実行してください。

開発環境または本番環境に応じて、クライアント設定で正しいエンドポイントを使うようにしてください。
