## ステップバイステップガイド

プログラムを構築し、MagicBlock の Delegation Program `DELeGGvXpWV2fqJUhqcF5ZSYMS4JTLjteaAMARRSaeSh` を使って delegation hooks を追加しアップグレードします。

<Steps>
  <Step title={<a href="#1-write-program">プログラムを書く</a>}>
    いつも通りに Solana プログラムを書きます。
  </Step>
  <Step
    title={
      <a href="#2-delegate">
        プログラムに delegation と undelegation hooks を追加する
      </a>
    }
  >
    CPI hooks を追加し、状態アカウントを Ephemeral Rollup セッション経由で delegate、commit、undelegate できるようにします。
    <Validators />
  </Step>
  <Step title={<a href="#3-deploy">プログラムを Solana にデプロイする。</a>}>
    Anchor または Solana CLI を使ってプログラムを Solana に直接デプロイします。
  </Step>
  <Step
    title={
      <a href="#4-test">
        delegation とリアルタイム高速取引の実行準備
      </a>
    }
  >
    SVM RPC 仕様に準拠したトランザクションを、変更なしでオンチェーン・オフチェーンの両方から送信します。
  </Step>
</Steps>

### コードスニペット

<Tabs>
  <Tab title="1. Write program">

    このプログラムは 2 つの主要 instruction を実装しています。

    1. `initialize`: カウンターを 0 に設定する
    2. `increment`: カウンターを 1 増やす

    また、カウンターの delegation と undelegation 用に次の instruction も実装しています。

    1. `Delegate`: カウンターを Base Layer から ER に委任する（Base Layer で呼ばれる）
    2. `CommitAndUndelegate`: カウンターを ER から Base Layer に同期するようスケジュールし、ER 上で undelegate する（ER で呼ばれる）
    3. `Commit`: カウンターを ER から Base Layer に同期するようスケジュールする（ER で呼ばれる）
    4. `Undelegate`:
        - カウンターの同期と undelegation をスケジュールする（ER で呼ばれる）
        - `#[ephemeral]` によって注入された callback instruction で undelegation が実行される（validator CPI を通じて Base Layer で呼ばれる）
    <UndelegationCallbackNote />
    <WriteProgramCode />

    特別なことはなく、単純な Anchor カウンタープログラムです。違いは、undelegation 用の `ephemeral` macro と、delegation program とやり取りするロジックを注入する `delegate` macro を追加していることです。

    [⬆️ Back to Top](#code-snippets)

  </Tab>
  <Tab title="2. Delegate">
      <Validators />
    1. Anchor 機能付き `ephemeral-rollups-sdk` をプログラムに追加する

    ```bash
    cargo add ephemeral-rollups-sdk --features anchor
    ```

    `delegate`、`commit`、`ephemeral`、`DelegateConfig`、`commit_accounts`、`commit_and_undelegate_accounts` をインポートします。

    ```rust
    use ephemeral_rollups_sdk::anchor::{
      commit,
      delegate,
      ephemeral
    };
    use ephemeral_rollups_sdk::cpi::DelegateConfig;
    use ephemeral_rollups_sdk::ephem::{
      commit_accounts,
      commit_and_undelegate_accounts
      };
    ```

    2. プログラムに `delegate` macro と instruction、`ephemeral` macro、`undelegate` instruction を追加します。auto commit や特定 ER validator など、好みの delegation config も指定できます。

  <Tab title="Resize PDA">
    委任済み PDA をリサイズする際には:
    - 新しいアカウントサイズでも rent-exempt を保てるだけの lamports が PDA に必要です。
    - 追加 lamports が必要な場合、差額を提供するために **payer account も委任済みである必要があります**。
    - PDA はプログラム所有である必要があり、lamports 転送に必要な signer をトランザクションに含める必要があります。
    - `system_instruction::allocate` を使用します。

  <Tab title="委任済みアカウントへの lamports チャージ">
    委任済みアカウントに lamports を追加チャージします。トランザクションは **base layer** で提出し、Ephemeral SPL Token プログラムが使い捨ての lamports PDA を介して委任済み残高へ lamports を送り込みます。

    主な用途：委任済み fee payer に資金を補充し、デフォルトで提供される 10 回の commit スポンサーシップ枠を超えても委任済み fee payer 自身で commit 費用を支払えるようにします。

    ポイント：
    - チャージごとに `crypto.getRandomValues` で新しい 32 バイトの salt を生成します — salt を使い回すと既存の PDA と衝突します。
    - ER ではなく **base layer の RPC** に提出します。
    - 送金先アカウントはすでに委任済みである必要があります。
