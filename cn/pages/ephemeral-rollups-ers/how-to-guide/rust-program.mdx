构建你的程序，并通过 MagicBlock 的 Delegation Program `DELeGGvXpWV2fqJUhqcF5ZSYMS4JTLjteaAMARRSaeSh` 加入 delegation hooks 完成升级：

<Steps>
  <Step
    title={
      <a href="#1-write-program">
        编写程序并添加 delegation instructions
      </a>
    }
  >
    按照你平常的方式编写 Solana 程序。
  </Step>
  <Step title={<a href="#2-delegate">在 Base Layer 上 Delegate PDA</a>}>
    添加 CPI hooks，使状态账户能够在 Ephemeral Rollup 会话中完成 delegate、commit 和 undelegate。
    <Validators />
  </Step>
  <Step title={<a href="#3-commit">在 ER 上 Commit PDA</a>}>

  </Step>
  <Step title={<a href="#4-undelegate">在 ER 上 Undelegate PDA</a>}>
    在链上和链下发送无需修改且符合 SVM RPC 规范的交易。
  </Step>
</Steps>

  <Tab title="2. Delegate">
    ### 委托 Counter PDA

    为了委托 counter PDA，并使其在 Ephemeral Rollup 会话中可写，我们需要添加一个 instruction，
    其内部会调用 `delegate_account` 函数。`delegate_account` 会通过 CPI 调用 delegation program，
    在验证通过后 delegation program 将获得该账户的所有权。
    完成这一步后，ephemeral validator 就可以开始在 counter PDA 上处理交易，并通过 delegation program 提交状态差异。

    ### 取消委托 PDA

    取消委托 PDA 是通过在某个 instruction 中调用 `commit_and_undelegate_accounts` 来完成的。
    undelegation 会提交最新状态，并将 PDA 的所有权归还给 owner program。完成 undelegation 和状态最终确认后，validator 会在 Base Layer 上触发一个对 `"undelegate"` 的 CPI 回调。
