### 랜덤니스에 block hashes를 사용하지 않는 이유는 무엇인가요?

Block hashes는 채굴자와 검증자에 의해 예측되거나 조작될 수 있어 공정한 게임플레이에 적합하지 않습니다. MagicBlock의 VRF는 랜덤니스가 block production과 독립적으로 생성되었음을 증명합니다.

### 랜덤니스 callback은 얼마나 빠른가요?

VRF는 [ephemeral rollup](/ko/pages/ephemeral-rollups-ers/introduction/ephemeral-rollup) execution window 안에서 동작하므로, 결과는 보통 외부 polling 없이 단일 transaction round 안에서 사용 가능합니다.

### 랜덤니스 proof를 감사할 수 있나요?

네. 각 request에는 누구나 VRF signer와 동일한 cryptography를 사용해 온체인에서 검증할 수 있는 proof가 포함됩니다. 이러한 투명성은 플레이어가 공정성을 확인할 수 있게 합니다.

### 오라클은 누가 운영하나요?

EphemeralVrf는 on-chain queues를 통해 관리되는 permissioned set of oracles를 사용합니다. rollup을 특정 queue에 위임하면 신뢰할 수 있는 운영자가 request를 처리하게 할 수 있습니다.
