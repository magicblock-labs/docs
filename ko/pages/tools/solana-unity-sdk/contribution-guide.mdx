## 작업할 이슈 찾기

시작하고 싶다면 먼저 [열려 있는 이슈](https://github.com/magicblock-labs/Solana.Unity-SDK/issues)를 확인하세요. 간단한 문서 수정이나 오타 수정은 바로 pull request를 열어도 좋습니다.

더 큰 변경이나 자발적인 기능 추가를 고려하고 있다면, 먼저 이슈를 등록하고 [Telegram](https://t.me/+78KHQkUsy0ViMzQ6)에서 메인테이너와 논의해 주세요.

## 이슈 선택하기

기여하고 싶다면 댓글을 남기고, 저장소를 포크한 뒤, 비어 있어도 괜찮으니 pull request를 열어 해당 이슈를 맡았다는 표시를 해 주세요. 이렇게 하면 메인테이너가 누가 어떤 이슈를 작업 중인지 추적할 수 있어 중복 작업을 막을 수 있습니다.

## 가이드라인

다음 가이드라인을 따라 주세요.

코딩 전:

- 작업 중인 이슈를 설명하는 브랜치 이름을 사용하세요

코딩 중:

- 가능한 한 빨리 draft PR을 제출하세요
- PR과 직접 관련된 코드만 수정하세요. 때로는 리팩터링이 꼭 필요해 보이는 코드를 발견할 수도 있지만, 현재 PR과 관련이 없다면 건드리지 말고 대신 이슈를 등록하세요. 이렇게 하면 리뷰어가 한 번에 하나의 문제에 집중할 수 있습니다.
