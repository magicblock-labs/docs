랜덤니스를 요청할 때 더 신뢰할 수 있는 결과를 얻으려면 아래 팁을 따르세요.
요청이 빠르게 처리되도록 `DelegateOracleQueue` 명령으로 rollup을 신뢰할 수 있는 오라클 큐에 위임하세요.

### 시드 선택

- 사용자 입력을 서버 타임스탬프나 게임 상태와 결합해 예측하기 어려운 시드를 만드세요.
- 결과를 반복 시도해 유리한 값을 찾는 일을 막기 위해 플레이어가 `caller_seed` 전체를 지정하지 못하게 하세요.

### 콜백 검증

- 콜백이 MagicBlock의 공식 서명자에서 왔는지 확인하기 위해 항상 `vrf_program_identity`를 검증하세요.
- 원래 요청 파라미터와 일치하지 않는 콜백은 거부하세요.

### 상태 관리

- 대기 중인 요청 식별자를 저장해 재시도나 예상치 못한 콜백을 안전하게 처리할 수 있게 하세요.
- 요청이 어떤 오라클 큐에 위임되었는지 추적해 결과를 올바르게 매칭할 수 있게 하세요.
- 랜덤 출력은 일회성으로 유지하고, [rollup 로직](/ko/pages/get-started/introduction/ephemeral-rollup) 안에서 즉시 소비하며 재사용하지 마세요.

이 가이드라인을 따르면 조작 위험을 줄이고 세션 전반에서 일관된 랜덤니스를 유지할 수 있습니다.
