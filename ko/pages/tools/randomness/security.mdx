title: "보안"
description: "랜덤성 proof가 어떻게 검증되는지"
---

랜덤성 proof는 입력 `caller_seed`와 MagicBlock의 VRF signer identity에 암호학적으로 묶여 있습니다. callback은 다음과 같이 이를 강제합니다.

```rust
#[account(address = ephemeral_vrf_sdk::consts::VRF_PROGRAM_IDENTITY)]
pub vrf_program_identity: Signer<'info>,
```

이 callback은 공식 MagicBlock oracle만 트리거할 수 있으므로, 위조되거나 조작된 결과를 막을 수 있습니다. 잘못된 proof는 자동으로 실패하며, 다른 programs가 이 요청을 front-run 할 수도 없습니다.

EphemeralVrf는 `InvalidProof`, `Unauthorized` 같은 조건을 검사하므로 잘못된 서명이나 미인가 caller는 게임 로직이 실행되기 전에 거부됩니다. 이 구현은 아직 감사를 받지 않았으므로, 실제 배포 전에 staging 환경에서 callback을 항상 모니터링해야 합니다.

사용자가 전체 `caller_seed`를 직접 제공하게 두지 말고, seed grinding을 방지하기 위해 게임 상태나 타임스탬프와 결합하세요. 모든 것이 결정론적 rollup 안에서 실행되므로 랜덤 값은 재사용되거나 지연될 수 없습니다.
이 검증은 게임 로직을 실행하는 것과 동일한 [ephemeral rollup](/ko/pages/get-started/introduction/ephemeral-rollup) 안에서 수행됩니다。
