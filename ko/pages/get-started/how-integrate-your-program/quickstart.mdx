통합 관점에서 Ephemeral Rollups를 사용하는 것은 멀티스레드 프로그램을 작성하는 것과 비슷합니다. 프로그램은 작업을 Ephemeral Rollup 세션으로 오프로드하여, 낮은 지연 시간과 높은 처리량을 가진 SVM runtime에서 트랜잭션을 처리할 수 있습니다. 이 SVM runtime은 필요한 시점에 즉시 프로비저닝되며, 트랜잭션은 탈중앙화된 ephemeral validators 네트워크의 노드에 의해 처리됩니다.

Ephemeral Rollups는 Solana 블록체인을 대상으로 하는 어떤 프로그램에도 사용할 수 있습니다. 우리는 다음과 같은 대표적인 통합 예제를 제공합니다.

## 통합 라이프사이클

프로그램에 Ephemeral Rollups를 통합하는 과정은 다음과 같습니다.

<Steps>
  <Step title="프로그램 작성">
    Anchor, native Rust, Bolt, C, 심지어 어셈블리까지, 평소처럼 Solana
    프로그램을 작성합니다.
  </Step>
  <Step title="delegation 및 undelegation hooks 추가">
    위임된 계정은 Ephemeral Rollup 세션에서 writable 계정으로 사용할 수
    있습니다.
  </Step>
  <Step title="프로그램을 Solana에 배포">
    프로그램을 Solana에 직접 배포합니다.
  </Step>
  <Step title="트랜잭션 실행">
    SVM RPC 사양을 따르는 SDK(web3.js, Solana Rust SDK, Unity SDK 등)를
    사용해 온체인 또는 오프체인에서 트랜잭션을 실행합니다.
  </Step>
</Steps>

## delegation 및 undelegation hooks 추가

delegation 및 undelegation hooks를 추가하면 프로그램에 초저지연 트랜잭션 능력을 부여할 수 있습니다. native Rust SDK의 함수 두 개만 프로그램에 추가하면 됩니다.

### Delegation

`Delegation` 은 프로그램의 하나 이상의 `PDA` 소유권을 delegation program으로 이전하는 과정입니다. 그러면 Ephemeral Validators가 이 `PDA` 를 사용해 SVM runtime에서 트랜잭션을 실행하고 상태를 커밋할 수 있습니다。

Rust 프로그램에서는 `ephemeral_rollups_sdk` crate를 사용해 계정을 위임할 수 있습니다.

다음과 같이 설치합니다.

```bash
cargo add ephemeral_rollups_sdk
```

그다음 `delegate_account` 함수를 사용해 계정을 delegation program에 위임합니다.

```rust Delegation
use ephemeral_rollups_sdk::cpi::delegate_account;

delegate_account(
    &ctx.accounts.payer, // The account that will pay for opening the session
    &ctx.accounts.pda, // The PDA to delegate
    &ctx.accounts.owner_program, // Owner program of the PDA
    ...
    pda_seeds, // Seeds to make the PDA signer
    0, // 0 means no time limit for the delegation
    3_000, // Update frequency on the base layer in milliseconds
)?;
```

<Note>
  delegation과 undelegation은 모두 CPI이며, 프로그램의 기존 instruction에
  통합할 수 있습니다
</Note>

### Undelegation

`Undelegation` 은 `PDA` 의 소유권을 다시 프로그램으로 되돌리는 과정입니다.
undelegation 시 상태가 커밋되고 finalization 프로세스가 시작됩니다. 상태 검증이 완료되면 `PDA` 가 잠금 해제되어 mainnet에서 평소처럼 사용할 수 있습니다.

```rust Delegation
use ephemeral_rollups_sdk::ephem::commit_and_undelegate_accounts;

pub fn undelegate(ctx: Context<IncrementAndCommit>) -> Result<()> {
    commit_and_undelegate_accounts(
        &ctx.accounts.payer,
        vec![&ctx.accounts.pda.to_account_info()],
        &ctx.accounts.magic_context,
        &ctx.accounts.magic_program,
    )?;
    Ok(())
}
```

추가로, custom CPI를 사용하면 ephemeral validators에 상태를 `commit`, `finalize` 하거나 세션을 `close` 하도록 지시할 수 있습니다.

<Note>
  commit와 undelegation은 여러 계정의 리스트를 받습니다. 이 계정들은
  베이스 레이어에 원자적으로 커밋되므로, 서로 의존하는 계정들의 상태 일관성을
  유지할 수 있습니다
</Note>

## 프런트엔드

프런트엔드 통합을 더 쉽게 하기 위해 [Magic Router](/ko/pages/get-started/how-integrate-your-program/magic-router)를 만들었습니다. 트랜잭션을 magic router로 직접 보내면, 그것이 [Ephemeral Rollup](/ko/pages/get-started/introduction/ephemeral-rollup)으로 가야 하는지 베이스 레이어로 가야 하는지를 자동으로 판단해 줍니다.

```typescript Frontend
import { sendMagicTransaction } from "magic-router-sdk";

const routerConnection = new Connection("https://devnet-router.magicblock.app", "confirmed");

// Construct a standard Solana transaction
const tx = await exampleClient.current?.methods
  .exampleMethod(0)
  .accounts({
      user: userPda,
      user: userKeypair.current.publicKey,
  }).transaction() as Transaction;

const noopInstruction = new TransactionInstruction({
      programId: new PublicKey('11111111111111111111111'),
      keys: [],
      data: Buffer.from(crypto.getRandomValues(new Uint8Array(5))),
  });
transaction.add(noopInstruction);

// Send the transaction
const signature = await sendMagicTransaction(
  routerConnection,
  tx,
  [userKeypair]
)
```
